# v0.1 — MVP read-only order-intent bot

## Назначение версии

v0.1 — базовый каркас проекта: бот для Bybit Spot, который анализирует публичные/read-only данные и пишет локальные торговые намерения в базу данных, но **не размещает ордера на бирже**.

Основной принцип:

```text
Market data → features → regime → strategy decision → signal/order intent ledger
```

Версия создавалась как безопасный foundation для последующих итераций. В ней важно было не максимизировать сложность стратегии, а заложить правильные границы ответственности, хранение решений, тестируемость и контейнерный запуск.

## Safety-инварианты v0.1

Обязательные ограничения, которые нельзя нарушать в будущих версиях:

```text
1. Нет методов place_order / cancel_order / amend_order.
2. Нет withdrawal/transfer операций.
3. Все торговые действия представлены как локальные записи в БД.
4. Bybit API keys, если используются, должны быть read-only.
5. Бот может писать BUY_INTENT/SELL_INTENT/CANCEL_INTENT/REPRICE_INTENT/HOLD, но не может исполнить их на бирже.
```

### 5. Market regime classifier

Файл:

```text
src/adaptive_bybit_bot/strategy/regime.py
```

Режимы:

```text
range
uptrend
uptrend_pullback
downtrend
shock
no_trade
```

Общий смысл:

```text
range              mean-reversion допустим при нормальном spread/volatility
uptrend            покупки только на откатах
uptrend_pullback   лучший кандидат для buy-limit
 downtrend          не ловить падающий рынок без отдельного подтверждения
shock              не торговать или сильно снизить активность
no_trade           рынок непригоден: плохие данные, spread, недостаточно свечей
```

### 6. Strategy engine

Файл:

```text
src/adaptive_bybit_bot/strategy/strategy.py
src/adaptive_bybit_bot/strategy/risk.py
src/adaptive_bybit_bot/strategy/order_pricing.py
```

Базовая логика:

```text
Нет позиции:
  - если режим и риск позволяют, создать BUY_INTENT;
  - если условия плохие, HOLD;
  - если старый intent устарел или edge исчез, CANCEL_INTENT/REPRICE_INTENT.

Есть позиция:
  - вычислить break-even с учётом комиссий;
  - создать SELL_INTENT выше break-even + target;
  - при ухудшении рынка запросить reprice/cancel/risk-exit;
  - при достижении max age / max loss формировать defensive exit.
```

Ключевые параметры из `.env.example`:

```env
ORDER_QUOTE_USDT=50
MAX_POSITION_QUOTE_USDT=250
SPOT_MAKER_FEE_BPS=10
SPOT_TAKER_FEE_BPS=10

SAFETY_BUFFER_BPS=5
MAX_SPREAD_BPS=8
MAX_ATR_PCT=1.4
SHOCK_ATR_PCT=1.2
MAX_UNREALIZED_LOSS_BPS=80
MAX_POSITION_AGE_SECONDS=7200
ORDER_TTL_SECONDS=120
REPRICE_THRESHOLD_BPS=4
MIN_EXPECTED_EDGE_BPS=30
```

## Известные ограничения v0.1

```text
1. Нет точных instrument filters: tickSize/qtyStep/minOrderAmt ещё не используются.
2. Paper-fill симулятор минимален.
3. Нет public WebSocket shadow loop.
4. Нет backtesting/replay.
5. Нет sentiment overlay.
6. Нет multi-service deployment.
7. Нет service locks/heartbeats.
8. Нет realistic fill model.
```

## Что нельзя ломать после v0.1

```text
- Read-only safety boundary.
- Order-intent ledger вместо прямой торговли.
- Явные reasons/metadata для каждого решения.
- Unit-тестируемость StrategyEngine без live Bybit.
- Возможность запуска на SQLite без Postgres.
```
