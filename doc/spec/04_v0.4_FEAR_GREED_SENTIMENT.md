# v0.4 — Alternative.me Fear & Greed sentiment overlay

## Назначение версии

v0.4 добавила sentiment-слой на базе Alternative.me Crypto Fear & Greed Index.

Ключевое архитектурное решение:

```text
Fear & Greed Index не является торговым сигналом.
Он является slow risk/aggressiveness overlay.
```

То есть индекс не должен сам говорить:

```text
BUY because Extreme Fear
SELL because Extreme Greed
```

Он должен менять параметры уже принятого решения:

```text
- required edge;
- размер intent;
- глубину лимитки;
- TTL;
- sell target;
- confidence multiplier;
- metadata/reason.
```

### 6. Symbol weights

Поскольку FNG — общий Bitcoin/crypto sentiment, а не отдельный индекс для каждой spot-пары, добавлены веса:

```env
FNG_BTC_WEIGHT=1.0
FNG_ETH_WEIGHT=0.6
FNG_ALT_WEIGHT=0.5
```

Ожидаемое поведение:

```text
BTCUSDT → modifiers применяются полностью.
ETHUSDT → modifiers применяются мягче.
Other    → default alt weight.
```

### 7. Strategy integration

Strategy engine теперь принимает optional sentiment context/policy.

Порядок применения:

```text
FeatureEngine
  ↓
RegimeClassifier
  ↓
StrategyEngine base decision
  ↓
SentimentPolicy modifies candidate parameters
  ↓
InstrumentSpec normalization/validation
  ↓
Signal + order intent persistence
```

Sentiment влияет на:

```text
required_buy_edge_bps
order_quote_usdt
buy limit distance
order ttl
sell target
confidence
metadata_json
reason_json
```

Sentiment не должен:

```text
- создавать BUY/SELL сам по себе;
- отменять safety/risk filters;
- разрешать торговлю в shock/no_trade режиме;
- игнорировать комиссии/spread.
```

### 8. Backtest integration

`backtest-csv` и `backtest-fetch` могут использовать cached FNG history, если:

```env
FNG_ENABLED=true
```

Перед offline backtest рекомендуется выполнить:

```bash
adaptive-bybit-bot fetch-fng --limit 30
```

Backtest должен подбирать sentiment context, доступный на timestamp свечи.

## Acceptance criteria v0.4

```text
1. fetch-fng сохраняет history в sentiment_fng.
2. list-fng показывает value/classification/timestamp/attribution.
3. /sentiment/fng отдаёт cached context.
4. Strategy metadata показывает применённый sentiment modifier.
5. FNG не генерирует сигнал сам по себе.
6. Backtest может использовать cached FNG values.
7. При FNG_DISABLED поведение стратегии остаётся как в v0.3.
```

Историческая проверка v0.4:

```text
34 passed
```

## Ограничения v0.4

```text
1. FNG обновляется медленно, daily-level; нельзя использовать как intraday trigger.
2. FNG primarily Bitcoin/crypto sentiment; для ETH используется reduced weight.
3. Нет специальных статистических отчётов по влиянию FNG.
4. Нет dashboard renderer с attribution.
5. Нет multi-service periodic sync до v0.5.
```

## Что нельзя ломать после v0.4

```text
- FNG_ENABLED=false должен полностью отключать overlay.
- Attribution должна сохраняться/отдаваться.
- FNG не должен обходить no_trade/shock/risk filters.
- Strategy unit tests должны оставаться deterministic без обращения к Alternative.me.
```
