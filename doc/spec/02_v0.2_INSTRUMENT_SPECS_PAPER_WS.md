# v0.2 — Instrument specs, paper-fill simulator и public WS client

## Назначение версии

v0.2 усилила MVP тем, что бот начал учитывать реальные торговые ограничения Bybit Spot перед созданием локального order intent.

Главная проблема v0.1:

```text
Стратегия могла рассчитать красивую limit price/qty, но такая заявка могла быть технически невозможна на бирже из-за tickSize, qtyStep или minimum notional.
```

Цель v0.2:

```text
Strategy decision → exchange filter validation → only valid order intent
```

Также в v0.2 были добавлены базовые элементы для последующего shadow/paper режима:

```text
- public WebSocket client;
- paper-fill simulator;
- сохранение paper fills;
- local positions update от paper/manual fills.
```

## Реализованный функционал

### 1. InstrumentSpec domain model

Добавлена модель:

```text
src/adaptive_bybit_bot/domain/models.py
InstrumentSpec
```

Поля:

```text
symbol
category
status
base_coin
quote_coin
price_tick_size
qty_step
min_order_qty
min_order_amount_quote
max_limit_order_qty
max_market_order_qty
base_precision
quote_precision
raw
```

Методы:

```python
InstrumentSpec.normalize_price(price, side)
InstrumentSpec.normalize_qty(qty)
InstrumentSpec.notional(price=..., qty=...)
InstrumentSpec.validate_limit_order(price=..., qty=...)
InstrumentSpec.fallback(symbol)
```

Правила округления:

```text
BUY price  → округлять вниз по tickSize
SELL price → округлять вверх по tickSize
qty        → округлять вниз по qtyStep/basePrecision
```

Причина:

```text
BUY не должен случайно пересечь ask из-за округления вверх.
SELL не должен продавать ниже рассчитанного target из-за округления вниз.
qty не должен превысить доступный/рассчитанный размер.
```

### 4. Strategy integration

Strategy engine теперь получает/использует `InstrumentSpec` при расчёте лимитки:

```text
raw buy/sell price
  ↓
normalize_price
  ↓
raw qty from ORDER_QUOTE_USDT / price
  ↓
normalize_qty
  ↓
validate_limit_order
  ↓
write signal/intent or HOLD with validation reason
```

Если в БД нет spec, используется fallback:

```python
InstrumentSpec.fallback(symbol)
```

Но production flow должен сначала выполнить:

```bash
adaptive-bybit-bot refresh-instruments --symbols BTCUSDT,ETHUSDT
```

### 6. Paper-fill simulator

Файл:

```text
src/adaptive_bybit_bot/services/paper_trading.py
```

Добавлены команды:

```bash
adaptive-bybit-bot paper-fill-once --symbol BTCUSDT
adaptive-bybit-bot paper-step --symbol BTCUSDT
adaptive-bybit-bot list-paper-fills
adaptive-bybit-bot list-positions
```

Настройки:

```env
PAPER_TRADING_ENABLED=false
PAPER_FILL_MODE=trade_through
PAPER_MIN_FILL_RATIO=1.0
PAPER_MAX_TRADE_AGE_SECONDS=300
```

Режимы fill approximation:

```text
trade_through:
  BUY fills, если later public trade price <= limit_price и volume достаточен.
  SELL fills, если later public trade price >= limit_price и volume достаточен.

touch:
  допускает fill, если книга touched/crossed limit.
```

Важное ограничение:

```text
Paper-fill не знает реальную позицию в очереди, hidden liquidity, partial fills и реальное место нашей лимитки.
```

### 7. Таблица `paper_fills`

Добавлена/активирована таблица:

```text

```

Поля:

```text

ts
order_intent_id
symbol
side

reason_json
```

При paper-fill:

```text
1. intent переводится в FILLED/PAPER_FILLED semantic state;
2. создаётся запись paper_fills;
3. обновляется локальная таблица positions;
4. создаётся order_event.
```

## Acceptance criteria v0.2

Версия считается корректной, если:

```text
1. refresh-instruments сохраняет specs для BTCUSDT/ETHUSDT.
2. StrategyEngine нормализует price/qty через InstrumentSpec.
3. Invalid local order intent не пишется как ACTIVE.
4. paper-fill-once может обработать активный intent на тестовых данных.
5. ws-print подключается к public WS и печатает сообщения.
6. Все тесты v0.1 остаются зелёными.
```

## Известные ограничения v0.2

```text
1. Public WS client ещё не поддерживает полноценную локальную книгу.
2. Paper-fill ещё слишком упрощённый.
3. Backtesting отсутствует.
4. Нет market recorder/replay.
5. Нет service locks/heartbeats.
6. Нет sentiment overlay.
```

## Что нельзя ломать после v0.2

```text
- InstrumentSpec.normalize_price/normalize_qty behavior.
- BUY округляется вниз, SELL округляется вверх.
- Не писать технически невозможные ACTIVE intents.
- Возможность работы без live Bybit в unit-тестах.
- Public WS остаётся unauthenticated.
```
