# v0.3 — WebSocket shadow mode, local orderbook и candle-level backtesting

## Назначение версии

v0.3 добавила первый серьёзный слой live-shadow и research tooling:

```text
Public WS stream → local cache/orderbook → strategy decisions → DB intents
Historical klines → candle-level backtest → persisted reports/fills
```

Цель версии:

```text
1. Использовать public WebSocket для более живого контекста рынка.
2. Научиться реконструировать локальную книгу по snapshot/delta.
3. Добавить offline backtest на свечах, чтобы быстро проверять параметры стратегии.
```

## Реализованный функционал

### 1. LocalOrderBook

Файл:

```text
src/adaptive_bybit_bot/market_data/orderbook.py
```

Основные классы:

```text
LocalOrderBook
LocalOrderBookStore
```

Правила обработки:

```text
snapshot:
  - полностью заменить bids/asks;
  - сбросить состояние sequence/update id;
  - отсортировать bids по убыванию price;
  - отсортировать asks по возрастанию price.

delta:
  - size == 0 → удалить price level;
  - size > 0 и level есть → update qty;
  - size > 0 и level отсутствует → insert level;
  - поддерживать sorted best bid/ask.

reset:
  - при новом snapshot;
  - при special update id u=1.
```

Публичные методы/свойства:

```text
best_bid
best_ask
spread_bps
mid
snapshot(depth=n)
apply_message(...)
```

### 4. WebSocket shadow strategy loop

Файл:

```text
src/adaptive_bybit_bot/services/ws_shadow.py
```

Команда:

```bash
adaptive-bybit-bot run-ws --symbols BTCUSDT,ETHUSDT
```

Логика:

```text
1. Подключиться к public Spot WS.
2. Подписаться на tickers/publicTrade/orderbook.
3. Поддерживать local orderbook/cache.
4. Периодически получать REST candles для FeatureEngine.
5. Собирать MarketSnapshot:
   - candles из REST;
   - orderbook/trades/ticker из WS cache.
6. Прогонять FeatureEngine → RegimeClassifier → StrategyEngine.
7. Писать signals/order_intents в БД.
```

Почему candles оставлены через REST:

```text
На v0.3 это проще и стабильнее, чем строить свечи из raw trades в live loop.
В v0.6 replay уже умеет агрегировать свечи из записанных trades.
```

### 6. Candle-level backtest engine

Файл:

```text
src/adaptive_bybit_bot/backtesting/engine.py
```

Команды:

```bash
adaptive-bybit-bot backtest-csv \
  --symbol BTCUSDT \
  --interval 1 \
  --input data/BTCUSDT-1m.csv

adaptive-bybit-bot backtest-fetch \
  --symbol BTCUSDT \
  --interval 1 \
  --start 2026-06-01 \
  --end 2026-06-02

adaptive-bybit-bot list-backtests
adaptive-bybit-bot list-backtest-fills --limit 100
```

Backtest model:

```text
- candles are replayed sequentially;
- strategy receives candle-derived synthetic MarketSnapshot;
- orderbook/spread approximated by synthetic spread bps;
- fills approximated at candle level;
- fees included;
- optional force close at end.
```

Настройки:

```env
BACKTEST_STARTING_QUOTE=10000
BACKTEST_WARMUP_CANDLES=240
BACKTEST_SYNTHETIC_SPREAD_BPS=2
BACKTEST_FORCE_CLOSE=true
```

## Acceptance criteria v0.3

Версия считается корректной, если:

```text
1. LocalOrderBook корректно применяет snapshot/delta.
2. delta size=0 удаляет level.
3. best bid/ask/spread обновляются после каждого события.
4. run-ws пишет signals/intents без private Bybit access.
5. download-klines создаёт CSV.
6. backtest-csv создаёт backtest run/fills.
7. API отдаёт backtests/backtest-fills.
8. Все тесты v0.2 остаются зелёными.
```

## Ограничения v0.3

```text
1. Backtest candle-level approximation: нет реальной intrabar sequence.
2. Нет raw market recorder.
3. Нет replay на recorded WS events.
4. Нет realistic queue/partial fill model.
5. Нет sentiment overlay.
6. Нет multi-service locks.
```

## Что нельзя ломать после v0.3

```text
- Корректную обработку orderbook snapshot/delta.
- Public WS должен оставаться без auth.
- run-ws не должен размещать ордера.
- Backtest должен быть deterministic на одинаковом CSV.
- Backtest persistence должна сохранять summary/fills.
```
